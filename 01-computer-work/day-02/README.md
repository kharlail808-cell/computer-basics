# Tujuan Belajar
Memahami hubungan CPU, RAM, dan Storage saat menjalankan aplikasi.
Mengamati penggunaan sumber daya komputer melalui Task Manager.

# Materi
CPU menjalankan instruksi.
RAM menyimpan data sementara.
Storage menyimpan data permanen.
Aktivitas Disk meningkat saat aplikasi dibuka.
Task Manager menampilkan penggunaan CPU seluruh sistem.

# Praktik
Tujuan
Mengamati perubahan penggunaan CPU, RAM, dan Disk saat membuka beberapa aplikasi.
Alat
Laptop
Windows
Task Manager
Langkah
1.Membuka Task Manager.
2.Mencatat kondisi awal.
3.Membuka WordPad.
4.Membuka Calculator.
5.Membuka File Explorer.
6.Membuka Google Chrome.
7.Mengamati perubahan CPU, RAM, dan Disk.


# Hasil Pengamatan
Aplikasi	CPU	RAM	Disk	Pengamatan
WordPad	32%	3,1 GB	1%	Aplikasi ringan, CPU dipengaruhi proses sistem.
Calculator	39%	3,1 GB	1%	Perubahan kecil pada RAM, CPU masih dipengaruhi proses lain.
File Explorer	37%	2,9 GB	1%	CPU naik saat memuat isi folder.
Google Chrome	27%	3,9 GB	0%	Menggunakan RAM paling banyak karena memuat halaman web.


# Analisis
Google Chrome menggunakan RAM paling banyak karena harus memuat halaman web dan menjalankan beberapa proses.
SSD paling aktif saat aplikasi pertama kali dibuka untuk membaca file aplikasi.
CPU berubah-ubah karena Task Manager menampilkan penggunaan CPU seluruh sistem, bukan hanya aplikasi yang sedang dibuka.

# Kesimpulan
Saya memahami bahwa CPU, RAM, dan Storage memiliki fungsi yang berbeda tetapi saling bekerja sama saat menjalankan aplikasi. Setiap aplikasi memiliki kebutuhan sumber daya yang berbeda, dan penggunaan CPU dapat dipengaruhi oleh proses lain yang berjalan di latar belakang.
