# Materi

## Hubungan CPU, RAM, dan Storage

Ketika sebuah aplikasi dibuka, CPU, RAM, dan Storage bekerja secara bersamaan.

### CPU (Central Processing Unit)

CPU bertugas menjalankan instruksi dan mengolah data sehingga aplikasi dapat digunakan.

### RAM (Random Access Memory)

RAM menyimpan data sementara yang sedang digunakan oleh aplikasi agar CPU dapat mengaksesnya dengan cepat.

### Storage (SSD)

Storage menyimpan data secara permanen, seperti sistem operasi, aplikasi, dan file pengguna. Saat aplikasi dibuka, SSD membaca data aplikasi sebelum dipindahkan ke RAM.

### Aktivitas Disk

Penggunaan Disk biasanya meningkat ketika aplikasi pertama kali dibuka karena Storage sedang membaca data yang diperlukan. Setelah aplikasi selesai dimuat, aktivitas Disk akan kembali menurun.

### Task Manager

Task Manager menampilkan penggunaan CPU, RAM, dan Disk dari seluruh sistem, bukan hanya dari aplikasi yang sedang dibuka.

# Analisis

Berdasarkan hasil praktik:

* Google Chrome menggunakan RAM paling banyak karena harus memuat halaman web, ekstensi, dan beberapa proses secara bersamaan.
* Aktivitas SSD paling tinggi saat aplikasi pertama kali dibuka karena Storage membaca data aplikasi yang diperlukan.
* CPU berubah-ubah selama praktik karena Task Manager menampilkan penggunaan CPU dari seluruh sistem, termasuk proses Windows yang berjalan di latar belakang.
* Setiap aplikasi memiliki kebutuhan sumber daya yang berbeda sehingga penggunaan CPU, RAM, dan Disk tidak selalu sama.

# Kesimpulan

Saya memahami bahwa CPU, RAM, dan Storage memiliki fungsi yang berbeda, tetapi saling bekerja sama ketika menjalankan sebuah aplikasi.

Urutan proses secara umum adalah:

1. Storage membaca data aplikasi.
2. Data dipindahkan ke RAM.
3. CPU menjalankan instruksi aplikasi.
4. Hasil ditampilkan pada monitor.

Selain itu, saya juga memahami bahwa penggunaan CPU, RAM, dan Disk dapat berubah-ubah tergantung aplikasi yang dijalankan serta aktivitas sistem operasi di latar belakang.
